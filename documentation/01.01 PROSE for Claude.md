## PROSE → Claude Code Translation Reference

| PROSE Primitive | Copilot Location | Claude Code Location | How It Works |
|---|---|---|---|
| **Global instructions** | `.github/copilot-instructions.md` | `CLAUDE.md` | Auto-loaded every session |
| **Domain instructions** | `.github/instructions/*.instructions.md` + `applyTo` | `.claude/rules/*.md` + `paths:` | Auto-loaded when editing matching files |
| **Chat modes (roles)** | `.github/chatmodes/*.chatmode.md` | `.claude/skills/*/SKILL.md` + `allowed-tools` | Invoked via `/architect`, `/frontend-engineer`, etc. |
| **Agentic workflows** | `.github/prompts/*.prompt.md` | `.claude/skills/*/SKILL.md` | Invoked via `/code-review`, `/debug`, etc. |
| **Spec templates** | `.github/specs/*.spec.md` | `.claude/skills/spec/SKILL.md` | Invoked via `/spec [feature-name]` |
| **Hierarchical context** | `AGENTS.md` in subdirectories | `CLAUDE.md` in subdirectories | Auto-discovered by walking directory tree |
| **Session memory** | `.memory.md` | Auto memory (built-in) | Claude persists learnings automatically at `~/.claude/projects/` |

## Key differences from the Copilot approach

- **No `.github/` directory needed** — Claude Code uses `CLAUDE.md` at project root and `.claude/` for configuration
- **Memory is built-in** — no `.memory.md` file needed; Claude auto-persists learnings across sessions
- **Skills replace both chat modes and prompts** — the `allowed-tools` frontmatter constrains tool access (like Copilot's MCP tool boundaries), while the skill body defines the workflow
- **Hierarchical context is native** — Claude walks up the directory tree and loads the closest `CLAUDE.md` automatically, just like `AGENTS.md`
